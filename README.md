# java-lotto-precourse

# 로또

## 기능 요구 사항
간단한 로또 발매기를 구현한다.
* 로또 번호의 숫자 범위는 1~45까지이다.
* 1개의 로또를 발행할 때 중복되지 않는 6개의 숫자를 뽑는다.
* 당첨 번호 추첨 시 중복되지 않는 숫자 6개와 보너스 번호 1개를 뽑는다.
* 당첨은 1등부터 5등까지 있다. 당첨 기준과 금액은 아래와 같다.
  * 1등: 6개 번호 일치 / 2,000,000,000원
  * 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
  * 3등: 5개 번호 일치 / 1,500,000원
  * 4등: 4개 번호 일치 / 50,000원
  * 5등: 3개 번호 일치 / 5,000원
* 로또 구입 금액을 입력하면 구입 금액에 해당하는 만큼 로또를 발행해야 한다.
* 로또 1장의 가격은 1,000원이다.
* 당첨 번호와 보너스 번호를 입력받는다.
* 사용자가 구매한 로또 번호와 당첨 번호를 비교하여 당첨 내역 및 수익률을 출력하고 로또 게임을 종료한다.
* 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException을 발생시키고, "[ERROR]"로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받는다.
  * `Exception`이 아닌 `IllegalArgumentException`, `IllegalStateException` 등과 같은 명확한 유형을 처리한다.
 
## 입출력 요구 사항

### 입력

* 로또 구입 금액을 입력 받는다. 구입 금액은 1,000원 단위로 입력 받으며 1,000원으로 나누어 떨어지지 않는 경우 예외 처리한다.
```text
14000
```
* 당첨 번호를 입력 받는다. 번호는 쉼표(,)를 기준으로 구분한다.
```text
1,2,3,4,5,6
```
* 보너스 번호를 입력 받는다.
```text
7
```

### 출력

* 발행한 로또 수량 및 번호를 출력한다. 로또 번호는 오름차순으로 정렬하여 보여준다.
```text
8개를 구매했습니다.
[8, 21, 23, 41, 42, 43] 
[3, 5, 11, 16, 32, 38] 
[7, 11, 16, 35, 36, 44] 
[1, 8, 11, 31, 41, 42] 
[13, 14, 16, 38, 42, 45] 
[7, 11, 30, 40, 42, 43] 
[2, 13, 22, 32, 38, 45] 
[1, 3, 5, 14, 22, 45]
```
* 당첨 내역을 출력한다.
```text
3개 일치 (5,000원) - 1개
4개 일치 (50,000원) - 0개
5개 일치 (1,500,000원) - 0개
5개 일치, 보너스 볼 일치 (30,000,000원) - 0개
6개 일치 (2,000,000,000원) - 0개
```
* 수익률은 소수점 둘째 자리에서 반올림한다. (ex. 100.0%, 51.5%, 1,000,000.0%)
```text
총 수익률은 62.5%입니다.
```
* 예외 상황 시 에러 문구를 출력해야 한다. 단, 에러 문구는 "[ERROR]"로 시작해야 한다.
```text
[ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.
```

- [x] 구매 금액을 입력받는 기능

  - [x] 번호가 중복이면 &rarr; `IllegalArgumentException`
- [x] 보너스 번호를 입력받는 기능
  - [x] 당첨 번호와 중복되면 &rarr; `IllegalArgumentException`

### 출력
- [x] 구매한 로또를 출력하는 기능

- [x] 당첨 통계를 출력하는 기능
  - [x] 몇개의 번호가 일치했는지, 그 개수가 몇개인지
  - [x] 총 수익률

- [x] 잘못된 값을 입력하면 예외처리 하는 기능
  - [x] 에러 메시지는 [ERROR]로 시작
  - [x] 에러가 발생한 위치부터 다시 입력받는 기능

- [x] 로또가 당첨되었는지 확인하는 기능
- [x] 수익률을 계산하는 기능

## 고려 예외 사항

### 구매 금액 관련
- [x] 1000원으로 나누어 떨어지지 않을때
- [x] 구매 금액이 너무 클때
- [x] 숫자가 아닐때
- [x] 공백 or null 일때

### 당첨 번호 입력 관련
- [x] 중복 번호 입력
- [x] 6개가 아닐때
- [x] 숫자가 아닐때
- [x] 공백 or null이 포함되어있을때
- [x] 범위가 맞지 않을때
- [x] 구분자가 쉼표가 아닐때

### 보너스 번호 입력 관련
- [x] 당첨 번호와 중복
- [x] 공백
- [x] 범위 초과
- [x] 숫자가 아님

## 클래스 및 역할 요약

### Application
* 프로그램의 진입점
* 모든 객체를 생성하고 종속성을 주입한 후 `LottoController.run()`을 호출하여 게임을 시작

### LottoController
* 로또 프로그램의 전체적인 진행 흐름을 제어
* 사용자로부터 입력을 받고, 유효하지 않은 입력에 대해 재입력을 요청하며 서비스 계층 로직을 호출

### LottoConverter
* 입력 형식 및 유효성 검사
* 당첨 번호 문자열을 `Lotto` 객체로, 로또 객체와 보너스 번호를 `WinningLotto` 객체로 변환  

### MoneyConverter
* 입력 형식 및 유효성 검사
* 구입 금액 문자열이 유효한 형식인지 확인하고, 유효하지 않은 입력에 대해 예외 처리

### Lotto
* 1~45 사이의 중복되지 않는 6개 로또 번호를 관리하는 도메인 객체.
* 도메인 내부 검증 로직 구현

### LottoGenerator
* 1~45 범위에서 중복 없는 6개의 숫자를 뽑아 정렬된 새 `Lotto` 객체를 생성

### LottoMoney
* 로또 구입 금액을 나타내며, 금액이 1,000원 단위인지 검증
* 구입 금액으로 발행할 수 있는 로또 개수와 총 상금 대비 수익률을 계산

### LottoResult
* 최종 당첨 통계와 수익률을 저장하는 객체

### PurchasedLottos
* 사용자가 구매한 모든 `Lotto` 객체 컬렉션을 관리  
* 당첨 로또를 기준으로 등수별 통계를 계산

### WinningLotto
* 당첨 번호와 보너스 번호를 포함합니다.  
* 보너스 번호의 중복 및 범위를 검증하고, 구매 로또의 등수를 판별

### LottoResultService
* `PurchasedLottos`와 `WinningLotto`를 사용해 당첨 통계를 계산하고 총 상금을 기반으로 최종 수익률을 산출하여 `LottoResult`를 반환

### LottoService
* `LottoMoney`를 받아 `LottoGenerator`로 필요한 수량만큼 로또를 생성하고 `PurchasedLottos`로 포장하여 반환

### LottoFormatter
* 로또 번호 리스트를 출력 형식으로 변환하고 상금 및 수익률 포맷을 제공

### Rank
* 로또 당첨 등수를 정의한 `Enum`  
* 일치 개수와 보너스 볼 일치 여부에 따라 등수를 결정하고 상금액을 제공

### InputView
* 콘솔을 통해 로또 구입 금액, 당첨 번호, 보너스 번호를 입력받아 문자열로 반환

### OutputView
* 구매 수량, 로또 번호, 당첨 통계, 최종 수익률을 사용자에게 출력
